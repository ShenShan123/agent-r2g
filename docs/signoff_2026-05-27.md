# Signoff Report — 2026-05-27

Snapshot of DRC / LVS / RCX results across the full `design_cases/` corpus
on nangate45 after the LVS rule deployment and signoff backfill of 2026-05-26
through 2026-05-27.

Authoritative source: each design's `reports/{drc,lvs,rcx}.json`. Aggregated by
`tools/aggregate_signoff_results.py` and frozen as `docs/signoff_snapshot_2026-05-27.json`.

LVS jumped from 0% to 85.3% in this campaign — upstream
The-OpenROAD-Project/OpenROAD-flow-scripts ships an empty
`flow/platforms/nangate45/lvs/` directory, so `make lvs` previously returned
"LVS not supported on this platform" for every nangate45 run. The skill now
bundles a working `FreePDK45.lylvs` (commit c5770d5) adapted from
[laurentc2/FreePDK45_for_KLayout](https://github.com/laurentc2/FreePDK45_for_KLayout).

The "stuck" status comes from a known KLayout deadloop in FreePDK45's polygon-op
rules. Distribution across rule lines:

| `*.lydrc` line | Designs stuck |
|----------------|---------------|
| 131 (`or` op) | 117 |
| 91 (`or` op) | 93 |
| 121 (`or` op) | 20 |
| 58 (boolean op) | 1 |
| 361 | 1 |

These designs have a valid GDS; LVS and RCX still pass. See
`r2g-rtl2gds/references/failure-patterns.md` → "KLayout DRC Stuck on `or`"
for the long-form pattern.
